# Excel 转 PDF 批量转换工具 - 需求设计

## 1. 业务背景

业务人员有大量 Excel 文件需要转换为 PDF 格式，当前手动操作效率低下。需要一个**零门槛**的工具，业务人员双击即可运行，无需安装 Python 或配置环境。

## 2. 核心需求

| 需求项 | 描述 |
|--------|------|
| 批量转换 | 自动扫描指定文件夹下的所有 Excel 文件（`.xls`, `.xlsx`） |
| 多线程处理 | 利用多线程并发转换，加速处理 |
| 异常处理 | 遇到损坏文件自动跳过，记录日志 |
| 空白页控制 | 仅打印有内容的区域，避免空白页 |
| GUI 界面 | 提供简洁的图形界面，支持选择文件夹和查看进度 |
| 独立 EXE | 打包为单文件可执行程序，双击即用 |

## 3. 内部讨论记录

### 讨论主题：技术方案选型

**产品经理**：用户是 Windows 业务人员，零技术背景。工具必须"双击即用"，界面简洁明了，能看到转换进度和结果。

**架构师**：关于 Excel 转 PDF 的核心引擎，有三个方案：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **A: `win32com` (COM 自动化)** | 保真度最高，完美保留图表/格式 | 仅限 Windows + 需安装 Office |
| **B: `openpyxl` + `reportlab`** | 跨平台，纯 Python | 需手动重建布局，复杂表格难搞 |
| **C: LibreOffice 命令行** | 跨平台，保真度较好 | 需安装 LibreOffice，体积大 |

**测试工程师**：需要考虑的边界场景：
- Excel 文件被加密/密码保护
- 文件正在被其他程序打开
- 包含宏的 `.xlsm` 文件
- 超大文件（百万行级别）
- 文件名包含特殊字符

**用户代言人**：业务人员电脑上一定安装了 Office，因为他们天天用 Excel。所以方案 A 是最佳选择。

### 最终决策
- **选择方案 A：`win32com` COM 自动化**
- 理由：目标用户是 Windows + Office 环境，保真度最高，能完美处理图表、条件格式等复杂场景
- GUI 使用 `tkinter`（Python 内置，无需额外依赖）
- 打包使用 `PyInstaller -F` 生成单文件 EXE

## 4. 假设清单

> [!IMPORTANT]
> 以下假设需要用户确认

1. **目标平台**：Windows 10/11，已安装 Microsoft Office（Excel）
2. **文件格式**：主要处理 `.xls` 和 `.xlsx` 格式
3. **输出位置**：PDF 输出到同目录下或用户指定的输出文件夹
4. **文件量级**：预计单次处理几百到几千个文件
5. **开发环境**：当前在 macOS 上开发，最终需在 Windows 上打包 EXE

## 5. 功能规格

### 5.1 GUI 界面
- 选择输入文件夹（Excel 源文件夹）
- 选择输出文件夹（PDF 目标文件夹，可选，默认同目录）
- "开始转换"按钮
- 进度条 + 实时日志显示
- 转换完成后显示成功/失败/跳过统计

### 5.2 转换引擎
- 使用 `win32com` 调用 Excel COM 接口
- 自动检测有内容的打印区域，避免空白页
- 支持 `.xls` / `.xlsx` / `.xlsm` 格式
- 多线程：默认 4 个工作线程（可调）

### 5.3 异常处理
- 损坏文件：记录日志并跳过
- 被占用文件：重试一次后跳过
- 密码保护文件：跳过并记录
- 所有异常写入 `conversion_log.txt`

### 5.4 性能目标
- 单文件转换：< 5 秒
- 1000 个文件：< 30 分钟（4 线程）
